# Промпт для AI: Исправить padding всех кнопок size="lg"

## Контекст
На странице `/partners` были исправлены внутренние отступы (padding) у кнопок размера `lg`, чтобы текст не прилипал к краям. Теперь нужно применить такие же исправления ко всем кнопкам размера `lg` на всем сайте.

## Что уже сделано

1. **Компонент Button** (`apps/frontend/src/components/ui/button.tsx`):
   - Размер `lg` имеет правильные значения: `px-10 py-4` (40px горизонтально, 16px вертикально)

2. **Глобальные CSS стили** (`apps/frontend/src/app/globals.css`):
   - Добавлены глобальные стили для всех кнопок размера `lg`:
   ```css
   button.h-14,
   a.h-14,
   button[class*="h-14"][class*="rounded-md"],
   a[class*="h-14"][class*="rounded-md"] {
     padding-left: 2.5rem !important;    /* 40px */
     padding-right: 2.5rem !important;   /* 40px */
     padding-top: 1rem !important;       /* 16px */
     padding-bottom: 1rem !important;     /* 16px */
   }
   ```

## Задача для AI

Проверь и убедись, что:

1. ✅ Все кнопки с `size="lg"` имеют правильные padding (40px горизонтально, 16px вертикально)
2. ✅ Стили применяются как к `<button>`, так и к `<a>` элементам (когда используется `asChild` prop)
3. ✅ Проверь следующие страницы/компоненты:
   - Главная страница (Hero секция)
   - Страница партнёров (`/partners`)
   - Страницы городов (`/cities/[slug]`)
   - Страницы услуг (`/services/[slug]`)
   - Формы (LeadForm)
   - Header (мобильное меню)
   - CTA секции
   - Geography компонент

4. Если найдешь кнопки, где padding не применяется:
   - Добавь более специфичные CSS селекторы
   - Или проверь, не переопределяются ли стили где-то еще

5. Проверь в браузере через DevTools, что все кнопки `size="lg"` имеют:
   - `padding-left: 40px` (2.5rem)
   - `padding-right: 40px` (2.5rem)
   - `padding-top: 16px` (1rem)
   - `padding-bottom: 16px` (1rem)
