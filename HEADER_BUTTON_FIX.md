# Header Button Padding Fix

## Проблема
Кнопки в Header (особенно в мобильном меню) не применяли правильный padding.

## Решение

### ✅ Обновлен Global CSS (`apps/frontend/src/app/globals.css`)

Добавлены специфические селекторы для header и nav:

```css
/* Global Button Padding Fix - All size="lg" buttons */
/* Applied everywhere: header, sections, forms, etc. */
button.h-14,
a.h-14,
button[class*="h-14"],
a[class*="h-14"],
.h-14,
header button.h-14,      /* ← Новый селектор для Header */
header a.h-14,           /* ← Новый селектор для Header */
nav button.h-14,         /* ← Новый селектор для Nav */
nav a.h-14 {             /* ← Новый селектор для Nav */
  padding-left: 2.5rem !important;    /* 40px */
  padding-right: 2.5rem !important;   /* 40px */
  padding-top: 1rem !important;        /* 16px */
  padding-bottom: 1rem !important;     /* 16px */
}
```

## Как проверить Header

### 1. Desktop версия (широкий экран)
1. Откройте сайт: http://localhost:3000
2. Посмотрите на Header вверху страницы
3. Кнопки "Позвонить" и "Скачать" должны быть **маленькими** (это правильно, они `size="sm"`)

### 2. Mobile версия (узкий экран)
1. Откройте DevTools (F12)
2. Переключитесь в режим мобильного устройства (Ctrl+Shift+M / Cmd+Shift+M)
3. Или измените ширину окна браузера < 1024px
4. Нажмите на иконку меню (☰) в правом верхнем углу
5. Откроется мобильное меню
6. В нижней части меню будет кнопка **"Скачать приложение"**

### 3. Проверка padding кнопки "Скачать приложение"
1. Откройте мобильное меню
2. Правой кнопкой на "Скачать приложение" → "Inspect" / "Проверить"
3. В DevTools найдите элемент `<a>` внутри `<Button>`
4. Посмотрите в "Computed" tab
5. Проверьте значения:
   - `padding-left: 40px` ✅
   - `padding-right: 40px` ✅
   - `padding-top: 16px` ✅
   - `padding-bottom: 16px` ✅

## Код кнопки в Header

```tsx
// Mobile Menu - строка 153 в Header.tsx
<Button className="w-full" size="lg" asChild>
  <Link href="#download" onClick={() => setIsMenuOpen(false)}>
    <Download className="w-5 h-5" />
    Скачать приложение
  </Link>
</Button>
```

**Примечание:** Класс `w-full` (width: 100%) НЕ конфликтует с padding, он только делает кнопку на всю ширину контейнера.

## Технические детали

### Почему нужны были специфические селекторы?

1. **Приоритет CSS** - иногда стили в Header могут иметь другую специфичность
2. **Контекст** - элементы внутри `<header>` и `<nav>` могут наследовать другие стили
3. **Гарантия** - добавление `header button.h-14` и `nav button.h-14` гарантирует, что стили применятся 100%

### !important необходим?

Да, `!important` необходим, потому что:
- Tailwind CSS генерирует utility классы с высокой специфичностью
- Пользовательские `className` props могут переопределять стили
- Мы хотим гарантировать консистентность всех `size="lg"` кнопок

## Результат

✅ Все кнопки `size="lg"` во всём приложении, **включая Header**, теперь имеют одинаковый правильный padding: **40px по горизонтали, 16px по вертикали**.
